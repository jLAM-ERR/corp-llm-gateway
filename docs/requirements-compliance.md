**Уровни детекции (калибровка ожиданий — важно для аудита):**
- **Tier-1 — детерминированно ✅:** структурные сущности с чек-суммами (ИНН/КПП/ОГРН/…),
  форматные секреты (JWT/PEM/API-key), и **перечислимые** словари (продукты, термины ПОД/ФТ,
  грифы конфиденциальности). Совпадение — по чек-сумме/паттерну/лемме, без модели.
- **Tier-2 — best-effort 🟡:** непомеченное «ноу-хау» / коммерческая тайна / проприетарный
  алгоритм в чистом коде. Ловит только LLM-оракул (Presidio/DeepPavlov здесь слепы). Гарантии нет.
- **Tier-3 — политикой/процессом:** новое и непомеченное. Управляется требованием маркировки
  (переводит в Tier-1), блокировкой чувствительных маршрутов, DLP-фингерпринтом, async-shadow +
  ревью человеком.

## Ключевой архитектурный вывод (обновлён)

**Раньше** детекция была делегирована LLM-оракулу целиком (нет хардкод-детекторов).
**Теперь** — **local-first каскад** (план 20260630, `ADR-003`):

1. **Stage 0** — классификатор payload: конфиг/лог → отказ до egress.
2. **Локальный «пол»** (детерминированный, ~6 ms p50 на CPU): `replace.md` + regex/checksum
   (`detectors/regex_checksum.py`) + двуязычный NER run-both-union (`detectors/dual_ner.py`) +
   лемма-газеттир (`rules/gazetteer.py`) + сплиттер идентификаторов (`sanitizer/segmenter/`).
3. **LLM-оракул — условный fallback**: вызывается **только по детерминированному попаданию в
   газеттир** (продукты/ПОД-ФТ/грифы), иначе не вызывается (латентный выигрыш). Порогов
   уверенности нет.
4. **Stage 5** — DLP-guard: пере-скан исходящего на canary/секреты → блок.

**Латентность (замер CP0, `scripts/bench_local_first.py`):** локальный путь p50 ≈ 5–7 ms против
7.4–15.6 s round-trip оракула — на порядки быстрее, и это ответ на конкурентный бенчмарк
`ru-llm-proxy` (который на DeepPavlov/BERT тяжелее на CPU).

Что усиливает защиту независимо от модели (без изменений):
- **Инвариант «оригиналы не утекают»** (M1-14) — расширен на новые поверхности: вывод
  сегментатора, тело отказа Stage-0, логи DLP-guard.
- **Гейт NEVER-полей** (`audit/invariants.py` + Vector VRL) — включая SIEM-sink.
- **Fail-closed** при отказах (матрица M4); Stage-0/Stage-5 — явный refuse, не fail-open.
